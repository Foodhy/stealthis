Prisma Schema Architecture

Models, relations, multi-file schemas, migrations, and the Prisma Client generation pipeline.

Project Structure
prisma/ ├── schema.prisma (Single-file schema) │ ├── datasource db (provider, url) │ ├── generator client (output config) │ └── model User { } (Tables & relations) ├── schema/ (Multi-file alternative) │ ├── user.prisma │ ├── post.prisma │ └── billing.prisma ├── migrations/ │ ├── 20260101_init/ │ │ └── migration.sql │ └── migration_lock.toml └── seed.ts (Dev seed data)
Mini Schema Example
datasource db { provider = "postgresql" url = env("DATABASE_URL") } generator client { provider = "prisma-client-js" } model User { id Int @id @default(autoincrement()) email String @unique name String? posts Post[] // one-to-many } model Post { id Int @id @default(autoincrement()) title String author User @relation(fields: [authorId], references: [id]) authorId Int }
Workflow
Define Schema → prisma migrate dev → prisma generate → Use typed client

Client Generation Pipeline

schema.prisma → Prisma Engine (Rust) → AST → DMMF → @prisma/client (Typed)
