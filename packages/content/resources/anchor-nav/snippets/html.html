Anchor Navigation

Scroll down to see the sidebar highlight the active section in real time.

Introduction

Anchor navigation (scroll-spy) is a pattern that keeps users oriented within long-form content pages — documentation, blog posts, legal pages — by highlighting the section currently in view.

This component uses the native IntersectionObserver API for efficient, paint-free scroll detection. There are no event listeners on the scroll event itself, which keeps performance budget low even on complex pages.

The sidebar sticks to the viewport as the user scrolls, providing persistent access to every section link. Clicking a link smooth-scrolls to the target without a visible hash change in the URL bar.

Tip: Combine this with a reading-progress bar (the thin line at the top of this page) to give users a sense of how far through the content they are.

Features

This implementation ships with several quality-of-life features out of the box.

  • Scroll-spy via IntersectionObserver — the topmost visible section is always highlighted, not just the last one to cross the threshold.
  • Smooth scroll on click — no jarring anchor jumps; the page animates fluidly to the target.
  • Reading progress bar — a fixed-top bar fills as the user approaches the end of the content.
  • Accessible markup — aria-current, aria-label, and semantic <nav> element.
  • Zero dependencies — pure HTML, CSS, and Vanilla JS.

Usage

Drop the sidebar HTML alongside your content and ensure each content section has a unique id. The sidebar links must use href="#section-id" to match. Then include the script — the observer wires itself up automatically.

The sidebar is positioned with CSS position: sticky inside a two-column grid layout. It will follow the user as they scroll without any JavaScript required for the sticking behavior.

For mobile screens, the sidebar collapses into a compact horizontal pill-row nav at the top of the content. The same JS wires up the active state for both orientations.

Note: For this demo the page content is contained inside a scrollable .content element rather than the window — the IntersectionObserver uses the content wrapper as its root.

Customization

All visual tokens are declared as CSS custom properties on :root so you can override them without touching the component CSS.

Change the accent color, indicator thickness, sidebar width, or transition timing — all from a single set of variables at the top of your stylesheet.

The observer's rootMargin controls when sections are considered "active". The default value of -30% 0px -60% 0px means a section is highlighted when its top edge is within the upper 30–40 % of the viewport — a comfortable reading zone.
