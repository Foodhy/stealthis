The Art of Scroll UX

Long-form content — documentation, blog posts, landing pages — benefits enormously from a well-placed back-to-top button. Rather than forcing users to scroll all the way back, a single click returns them instantly. The key is subtlety: the button should only appear when it is needed and should never obscure content while reading.

Why Position: Fixed

The button uses position: fixed anchored to the bottom-right corner of the viewport. This keeps it consistently reachable regardless of scroll position, layout width, or device. A small margin from the edges prevents it from hugging too close to the browser chrome on mobile.

Smooth Scroll Performance

Using window.scrollTo({ top: 0, behavior: 'smooth' }) delegates animation to the browser's native compositor, which can run off the main thread. This avoids the jank of JS-driven scroll loops while still providing a polished, animated experience across all major browsers.

The Progress Ring

This variant adds a circular SVG progress ring around the button that fills as you scroll down the page. The ring is drawn with a stroke-dashoffset updated on every scroll frame. The circumference of the circle is pre-calculated and stored as a CSS custom property so the math stays in JavaScript but the animation stays in CSS.

Visibility Threshold

The button appears after 300 px of scroll — roughly one viewport height on a phone. This threshold prevents the button from flashing in for short pages. You can adjust it by changing the THRESHOLD constant in the script. For very long pages, a higher threshold (500–800 px) may feel more appropriate.

Accessibility Considerations

The button is always in the DOM so its focus state is predictable. It uses opacity: 0 and pointer-events: none when hidden — not display: none — which means keyboard focus can still land on it. An aria-label provides a meaningful name for screen readers, and the tabindex is left at the default 0.

Customization Tips

All visual tokens live in CSS custom properties: button size, accent color, offset from the edges, and transition timing. Override them at the :root level or scope them to a parent element. The progress ring radius and stroke width are derived from --btn-size automatically so the ring always fits the button perfectly.
