## React Native Text Input

A polished, reusable text input component for React Native featuring a smooth floating label animation, built-in validation states, helper text, and icon support. Zero external dependencies — built entirely with React Native's `Animated` API.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `label` | `string` | — | Floating label text displayed above or inside the input |
| `value` | `string` | — | Current input value |
| `onChangeText` | `(text: string) => void` | — | Callback fired when the text changes |
| `error` | `string` | `undefined` | Error message string; triggers the error visual state |
| `success` | `boolean` | `false` | When `true`, shows the success border color |
| `helperText` | `string` | `undefined` | Hint text rendered below the input |
| `leftIcon` | `ReactNode` | `undefined` | Element rendered on the left side of the input |
| `rightIcon` | `ReactNode` | `undefined` | Element rendered on the right side of the input |
| `secureTextEntry` | `boolean` | `false` | Masks the input for password fields |
| `disabled` | `boolean` | `false` | Disables interaction and dims the component |
| `...rest` | `TextInputProps` | — | All standard React Native `TextInput` props are forwarded |

### Usage

```tsx
import StyledTextInput from "./StyledTextInput";

function MyForm() {
  const [email, setEmail] = useState("");

  return (
    <StyledTextInput
      label="Email"
      value={email}
      onChangeText={setEmail}
      error={email.length > 0 && !email.includes("@") ? "Invalid email" : undefined}
      keyboardType="email-address"
      autoCapitalize="none"
    />
  );
}
```

### How it works

1. **Floating label** — The label starts centered inside the input. On focus or when a value is present, it animates upward and scales down using `Animated.timing`. The animation runs at native speed via `useNativeDriver` where supported.

2. **Border color transitions** — An `Animated.Value` drives the border color, interpolating between the default, focused (brand blue), error (red), and success (green) states for a smooth visual transition.

3. **Validation states** — Pass an `error` string to activate the error state (red border + message below the input) or set `success` to `true` for a green confirmation border. Both states override the default focus color.

4. **Icon slots** — `leftIcon` and `rightIcon` accept any `ReactNode`, so you can drop in vector icons, images, or custom components. The input padding adjusts automatically.

5. **Accessibility** — The component forwards all `TextInput` props, including `accessibilityLabel`, `accessibilityHint`, and `testID`, so it integrates seamlessly with testing and screen-reader tooling.
