Key Principles

Dependency Inversion

Inner layers define interfaces; outer layers implement them. Domain never imports Express or Prisma.

Testability

Business logic is unit-tested without databases, HTTP, or any I/O. Mock at the boundary.

Framework Independence

Swap Express for Fastify, Prisma for Drizzle — domain and use cases remain untouched.

Screaming Architecture

Folder structure tells you what the system does, not what framework it uses.

The Dependency Rule

Frameworks → Adapters → Use Cases → Entities

Source code dependencies must always point inward, toward higher-level policies.
