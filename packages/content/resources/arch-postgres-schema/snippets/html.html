PostgreSQL Schema Patterns

Multi-schema organization, indexing strategies, migration workflows, and schema-as-code with Drizzle.

Project Structure
drizzle.config.ts (Connection + migrations) src/db/ ├── index.ts (Client export) ├── schema/ │ ├── auth.ts (users, sessions, accounts) │ ├── billing.ts (plans, subscriptions, invoices) │ ├── content.ts (posts, comments, tags) │ └── index.ts (Re-export all schemas) ├── migrations/ (Auto-generated SQL) │ ├── 0000_init.sql │ └── 0001_add_billing.sql └── seed.ts (Development data)
Multi-Schema Pattern

Domain-Driven Schema Files

auth.ts users, sessions, accounts
billing.ts plans, subscriptions, invoices
content.ts posts, comments, tags
Indexing Strategies

B-tree

Default. Equality and range queries.

CREATE INDEX ON users(email)

GIN

Full-text search, JSONB, arrays.

CREATE INDEX ON posts USING gin(tags)

Partial

Conditional subset of rows.

... WHERE deleted_at IS NULL

Composite

Multi-column for common queries.

... ON orders(user_id, created_at)
Migration Workflow
Edit schema.ts → drizzle-kit generate → Review SQL → drizzle-kit migrate → Commit
