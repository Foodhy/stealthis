## React Native Drawer Navigation

A fully custom slide-out drawer navigation component built with pure React Native primitives. No third-party navigation libraries required — just `Animated`, `PanResponder`, and standard RN views.

The drawer slides in from the left edge with a spring animation, dims the main content behind a translucent overlay backdrop, and supports both swipe gestures and tap-to-close interactions.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | — | The main screen content rendered behind the drawer. |
| `drawerContent` | `ReactNode` | Built-in menu | Optional custom content to render inside the drawer panel. |
| `drawerWidth` | `number` | `280` | Width of the drawer panel in pixels. |

### Usage

```tsx
import DrawerNavigator from "./DrawerNavigator";

// With default drawer content
<DrawerNavigator>
  <YourMainScreen />
</DrawerNavigator>

// With custom drawer content
<DrawerNavigator drawerContent={<CustomMenu />}>
  <YourMainScreen />
</DrawerNavigator>
```

### How it works

1. **PanResponder** captures horizontal swipe gestures starting from the left 30px edge of the screen. A rightward swipe beyond a velocity or distance threshold opens the drawer; a leftward swipe closes it.
2. An `Animated.Value` drives both the drawer's `translateX` and the overlay's opacity simultaneously, keeping them perfectly in sync.
3. The overlay backdrop uses `Animated.timing` with a spring-like configuration for a natural feel when opening, and a shorter ease-out when closing.
4. Tapping the overlay fires `onPress` to close the drawer, giving users two intuitive ways to dismiss it (swipe or tap).
5. The built-in drawer content renders a user profile header, a list of menu items with emoji icons and an active-state highlight, and a separated logout action at the bottom.
