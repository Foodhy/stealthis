## React Native Badge

A lightweight badge and chip component system for React Native. Badges display short status labels with semantic color variants, while chips act as interactive, pressable pills that can be selected or dismissed.

### Variants

Five color variants are provided — `default`, `success`, `warning`, `error`, and `info`. Each variant maps to a background/text color pair that works well on dark surfaces.

### Dot Mode

When the `dot` prop is enabled, the badge renders as a small colored circle without any text label — useful for compact status indicators.

### Props

**Badge**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `label` | `string` | — | Text content of the badge |
| `variant` | `"default" \| "success" \| "warning" \| "error" \| "info"` | `"default"` | Color variant |
| `size` | `"sm" \| "md"` | `"md"` | Badge size |
| `dot` | `boolean` | `false` | Render as a dot indicator only |

**Chip**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `label` | `string` | — | Text content of the chip |
| `onPress` | `() => void` | — | Called when the chip is pressed |
| `onDismiss` | `() => void` | — | Shows a dismiss button; called when tapped |
| `icon` | `ReactNode` | — | Optional leading icon element |
| `selected` | `boolean` | `false` | Whether the chip is in a selected state |

### Usage

Import `Badge` and `Chip` and place them in your layout. Badges are non-interactive labels suited for status or count indicators. Chips are interactive and support press, selection, and dismissal.

### How it works

Badge colors are stored in a variant map. The `dot` flag swaps the rendered content for a small circle of the variant color. Chips use `Pressable` for touch handling and conditionally render a dismiss "X" button when `onDismiss` is provided. The `selected` state applies a highlighted border and background tint.
