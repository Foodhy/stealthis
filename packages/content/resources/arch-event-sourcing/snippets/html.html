Architecture

Event Sourcing + CQRS

Append-only event store, aggregate roots, projections, and read/write separation.

Project Structure
src/ domain/ ├── events/ (OrderCreated, ItemAdded, OrderShipped) ├── aggregates/ (OrderAggregate — applies events) └── commands/ (CreateOrder, AddItem, ShipOrder) application/ ├── command-handlers/ (Validate → Load → Apply → Store) └── query-handlers/ (Read from projections) infrastructure/ ├── event-store/ (Append-only log — PostgreSQL/EventStoreDB) ├── projections/ (OrderSummary, UserActivity read models) └── snapshots/ (Periodic state snapshots) read-models/ ├── order-list.ts (Queryable view) └── analytics.ts (Aggregated data)

CQRS Flow
Write Command → Handler → Aggregate → Event Store → Event Bus
↓
Read Query ← Read Model ← Projection ← Event Bus
Key Concepts

Event Store

Append-only log of immutable events. State is derived by replaying events. Complete audit trail.

Aggregates

Consistency boundaries that receive commands, validate rules, and produce events.

Projections

Disposable read models built from events. Delete and rebuild any time by replaying.

Snapshots

Periodic state cache. Restore snapshot + replay only recent events for fast loading.
