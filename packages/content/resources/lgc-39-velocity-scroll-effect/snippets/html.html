Velocity-Aware Scroll

Scroll faster to see speed lines intensify. The canvas layer reacts to scroll velocity — the faster you scroll, the stronger the visual feedback.

Scroll Speed Detection

Lenis provides real-time velocity data through its scroll event. We map that velocity to visual intensity — opacity, line count, and blur all scale with how fast you scroll.

Slow scrolling produces minimal effect. Fast scrolling creates a dramatic speed-lines tunnel on the Canvas overlay.

Canvas Speed Lines

A Canvas 2D layer sits fixed behind the page. On each animation frame, lines radiate from the center at lengths proportional to the scroll velocity magnitude.

The visual effect is inspired by motion blur in photography and the speed lines used in action manga. Faster scroll = more lines, higher opacity, longer streaks.

Parallax Intensity

Parallax layers in the background also react to velocity. Faster scrolling temporarily increases the parallax offset multiplier, creating a sense of dimensional acceleration through space.

This effect is commonly used in gaming landing pages, sportswear brands, and esports tournament sites.

Performance

The effect is optimized for 60fps using:

  • Canvas clearRect + redraw on each rAF tick (no DOM mutations)
  • Velocity decay — speed lines fade naturally when scrolling slows
  • Line count cap to prevent overdraw on slow devices
  • Disabled entirely if prefers-reduced-motion is active

Implementation Details

Access Lenis velocity via lenis.on('scroll', ({ velocity }) => ...). The velocity value is a float between roughly -30 and +30 depending on scroll speed.

Map the absolute value to your visual parameters:

  • lineCount = Math.min(maxLines, Math.abs(velocity) * 5)
  • lineLength = Math.abs(velocity) * 20 pixels
  • opacity = Math.min(0.8, Math.abs(velocity) * 0.05)
