Screen Reader Only Utilities

CSS patterns for visually hidden content that remains accessible to assistive technology.

Decorative vs Informative Images

alt text

Decorative images use alt="" and aria-hidden="true". Informative images need descriptive alt text.

Decorative alt="" aria-hidden="true"
Informative alt="Bar chart showing..."
Screen reader sees: Decorative: (skipped). Informative: "Bar chart showing monthly revenue increasing from $12k in January to $28k in June"

CSS Techniques Comparison

reference

Different CSS methods for hiding content and their effect on screen readers.

SR reads

.sr-only (clip-rect)

.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}

The gold standard. Content is invisible but accessible.

SR reads

.sr-only (clip-path)

.sr-only-alt {
  position: absolute;
  clip-path: inset(50%);
  width: 1px;
  height: 1px;
  overflow: hidden;
  white-space: nowrap;
}

Modern alternative using clip-path. Same result.

SR skips

display: none

.hidden {
  display: none;
}

Hides from everyone, including screen readers. Do not use for accessible hiding.

SR skips

visibility: hidden

.invisible {
  visibility: hidden;
}

Also hides from screen readers. Element still takes up space.

Visible on focus

.sr-only-focusable

.sr-only-focusable:focus,
.sr-only-focusable:active {
  position: static;
  width: auto;
  height: auto;
  clip: auto;
  clip-path: none;
  white-space: normal;
  overflow: visible;
  margin: 0;
}

Extends .sr-only. Becomes visible when focused (skip links).

Depends

aria-hidden="true"

<span aria-hidden="true">
  Decorative content
</span>

Hides from screen readers but remains visually visible. Use for decorative elements.
