## Expo Push Notifications

A complete local push notification setup using `expo-notifications`. This component handles permission requests, schedules notifications with configurable delays, and maintains a notification inbox that logs every received notification in the foreground.

### Features

- **Permission handling** — requests notification permissions on mount and shows the current authorization status.
- **Local scheduling** — schedule notifications immediately or with a delay (5s, 10s) using trigger-based scheduling.
- **Foreground listener** — captures notifications while the app is in the foreground via `addNotificationReceivedListener`.
- **Tap listener** — responds to notification taps with `addNotificationResponseReceivedListener`.
- **Inbox UI** — displays a scrollable log of all received notifications with timestamps.

### Usage

Install the dependency and drop the component into your app:

```bash
npx expo install expo-notifications
```

The demo renders three trigger buttons and a live notification inbox. Tap a button to schedule a notification, then watch it appear in the inbox when it fires.
