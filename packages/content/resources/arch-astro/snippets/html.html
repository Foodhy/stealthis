Architecture

Astro Project

Content-driven framework with island architecture, content collections, and file-based routing

Folder Structure
├── astro.config.mjs (Integrations, adapters) ├── tsconfig.json ├── public/ (Static assets) └── src/ ├── pages/ (File-based routing) │ ├── index.astro │ └── blog/[slug].astro ├── layouts/ (Page shells) │ └── Base.astro ├── components/ (Astro + React/Vue/Svelte islands) ├── content/ (Content collections) │ ├── config.ts (Schema definitions) │ └── blog/ ├── styles/ (Global CSS) └── i18n/ (Translations)
Island Architecture

Zero JS by Default

All pages render to static HTML at build time. No JavaScript is shipped unless you explicitly add interactive islands.

Static HTML

Selective Hydration

Interactive components hydrate independently using directives: client:load, client:idle, client:visible, or client:media.

Islands
Content Collections

Type-Safe Querying

Define Zod schemas in config.ts. Astro validates frontmatter at build time and provides fully typed query results.

content/config.ts

Markdown & MDX

Write content in Markdown or MDX with full frontmatter support. Content is queried with getCollection() and getEntry().

.md / .mdx
Integrations & Rendering

Integrations

Add React, Vue, Svelte, Tailwind, MDX, or Sitemap with one-line integrations. Each framework works as island components.

@astrojs/*

SSG vs SSR

Defaults to static generation. Enable SSR per-page with prerender = false or globally with an adapter (Node, Cloudflare, Vercel).
