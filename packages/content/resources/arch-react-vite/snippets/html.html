Architecture

React + Vite

Feature-based project structure with native ESM dev server and Rollup production builds

Folder Structure
├── index.html (Vite entry point) ├── vite.config.ts (Plugins, aliases, build) ├── tsconfig.json ├── public/ (Static assets) └── src/ ├── main.tsx (React root mount) ├── App.tsx (Root component) ├── components/ (Shared UI) ├── features/ (Feature modules) │ ├── auth/ │ └── dashboard/ ├── hooks/ (Custom hooks) ├── lib/ (Utilities) ├── styles/ (Global CSS) └── types/ (TypeScript types)
Key Files
index.html Vite's true entry point; contains the root div and module script tag
vite.config.ts Plugins, resolve aliases, dev server proxy, and build options
main.tsx React root mount with createRoot().render()
features/ Self-contained domain modules with co-located components and hooks
Dev Server (ESM) & Production Build (Rollup)

Dev Server

Native ES modules served on demand. No bundling step. Browser requests trigger on-the-fly transforms via esbuild. Instant HMR regardless of app size.

Native ESM

Production Build

Rollup bundles optimized output with tree-shaking, code-splitting, and asset hashing. CSS is extracted and minified automatically.

Rollup
Path Aliases

vite.config.ts

Define aliases in resolve.alias to map @ to ./src. Supports any prefix pattern for clean imports across the project.

resolve.alias

tsconfig.json

Mirror the same aliases in compilerOptions.paths so TypeScript resolves types correctly and editors provide autocomplete.
