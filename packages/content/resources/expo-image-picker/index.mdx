## Expo Image Picker

A versatile image picker component built with `expo-image-picker` that supports gallery selection, camera capture, multi-select, and a responsive image preview grid with individual removal. Drop it into any Expo project for a polished image selection experience.

### Features

- **Gallery selection** — Pick one or multiple images from the device photo library.
- **Camera capture** — Take a new photo directly from within the component.
- **Multi-select support** — Select multiple images at once from the gallery.
- **Image preview grid** — View all selected images in a responsive grid layout.
- **Remove individual images** — Tap the remove button on any image to deselect it.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `onImagesSelected` | `(uris: string[]) => void` | — | Callback fired whenever the selected images change. |
| `maxSelection` | `number` | `10` | Maximum number of images allowed. |
| `quality` | `number` | `0.8` | Image compression quality (0 to 1). |
| `allowsEditing` | `boolean` | `false` | Whether to show the editing UI after single selection. |

### Usage

```tsx
import ImagePickerGrid from "./ImagePickerGrid";

<ImagePickerGrid
  onImagesSelected={(uris) => console.log("Selected:", uris)}
  maxSelection={5}
/>
```

### How it works

1. The component renders two action buttons: "Choose from Gallery" and "Take Photo".
2. Pressing "Choose from Gallery" calls `launchImageLibraryAsync` with `allowsMultipleSelection` enabled, letting users pick several images at once.
3. Pressing "Take Photo" calls `launchCameraAsync` to capture a single new photo.
4. Selected images are stored in state and rendered in a two-column grid.
5. Each image in the grid displays a remove button in the top-right corner. Tapping it filters the image out of the selection.
6. The `onImagesSelected` callback fires whenever the image list changes, keeping the parent component in sync.
